Project Idea

Overview

My project is a mobile app that simulates playing with cards. It is meant to be played with a bunch of friends. The player who starts a game using the app is the dealer and players can join the game via a code. The game will have a table viewed from the top looking down and a classic deck of 52 cards. Cards can be dealt to each player automatically and players can drag and drop their cards wherever they want on the table and in any orientation (face up, face down, tilted and ect).

Motivation

As I have said in "About Me", I love playing video games and playing tabletop games and card games. I want to make something that combines both my love video games and tabletop/card games. Whenever I’m at a party, either me or my friends would ask, “do any of you bring cards?” and so I aim to create an app that allows people to play cards anywhere and anytime they want. There are many mobile card games involving a standard pack of 52 cards such as solitaire, poker and ect, however they all have fixed rules. I have noticed that when people play card games, they tend to play slightly modified versions with “house rules." This game allows players to play cards however they want.

Description

The app is a mobile game let’s players play with a standard deck of 52 cards. When the app is opened, there will be a “Start Game” button, a “Join Game” button, a “Decks” button and a “Shop” button.

Pressing on the “Deck” button will open up an inventory of all the different decks a player has. All the decks will be standard 52 decks, but they’ll be in different styles and aesthetics. Each player will start with one deck in the classic, regular style and as they play games, they earn coins to buy different style decks in the shop.

The person who starts the game is the dealer and other people can join the game by pressing “Join Game” and entering a code provided by the dealer similar to Kahoot.

Once a game starts, there will be a table viewed from the top looking down surrounded by icons of several provided avatars which represents the players.

The dealer can deal cards to all the players, they can allocate and distribute the cards however they want. Tapping on the icons will show how many cards that player has and that player is able to decide whether they wish to reveal any of their cards or not. Players can drag and drop cards anywhere they want on the table and in any orientation whether it is face up, face down, tilted and ect. This allows players to play any type of card games however they like. Players can return cards back to the deck through dragging and dropping them into the deck. There is a reset button that returns all cards back to the dealer and a shuffle button which randomly shuffles the deck. The shuffle button will only shuffle cards in the deck that have not been dealt to players.

When players drag and move cards around, other players will not be able to see that card move around until the card is dropped on a new location in which then other players will see an animation of that card sliding to its new location. When players drag a card to another card, it will be stacked on top.

The maximum number of players will be 10 since the game, although can be played on a larger device, is intended to be played on a phone which has a small screen that may not fit all the icons representing the players.

Tools and Technology

I will be using a game engine to create my project. There are many game making engines for me to use and I believe the two best ones to use for my project are Unity and Godot. Game engines make game development much easier as it provides many built in features such as physics, 3d rendering and collision detection. This way developers can focus more on mechanics of the game instead of starting from scratch, although a bit of coding will be required.

Unity is one of the most popular game engines. As long as I earn less than 100k in the past 12 months, Unity is free to use. It specialises in making 3d games but can also make 2d games. Although not as complex and difficult to use as other game engines such as Unreal Engine, Unity still has a large learning curve. There are many features in Unity and while it allows high quality games to be made, it can be very challenging to learn all the features. I would have to learn a new programming language, C#.

Godot is another option I can use, and although not as good as Unity, it is free and open source. Godot is also easier to learn and use than Unity as you code using GDScript which is simpler than the language used in Unity (C#). Godot specialises in smaller 2d games which is what my project is.

Both Godot and Unity can run on Windows 10 which I have and I will be using my PC to run one of these game engines.
